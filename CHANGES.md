# データの引越し屋さん LP — リファイン変更点サマリー

方針：「確かな洗練 × デザインとコンバージョンの両立」。既存の世界観・制約はすべて維持し、完成度と無料診断への導線を引き上げました。単一HTML・外部JSなし（Google Fontsのみ）の形は変えていません。

---

## 追加改訂：文言のブラッシュアップ（競合リサーチ＋SEO/CRO一般論を反映）

- ねらい：冗長・説明過多で「AIっぽい」表現を排し、非ITの経営者に数秒で伝わる短く直截な文言へ。競合各社の実際の文章は複製せず、共通する「型・言い回しの慣習・検索キーワード」を抽出して独自コピーに反映しました（著作権に配慮した合法的な手法です）。
- 競合・市場リサーチ：日本の基幹システム移行・レガシー刷新・オフコン入れ替え関連の記事/LPを横断確認。頻出検索語（基幹システム、移行／入れ替え／刷新、レガシー、オフコン、保守終了、クラウド、中小製造業）と、定番の訴求順序（課題提起→放置リスク→一気通貫の解決→選ぶ理由→FAQ→CTA）を参照しました。
- CRO一般論：ファーストビューで「誰の・何の課題か」を即提示、1文を短く、単一の低摩擦CTA（無料診断）、CTA近傍で不安を先回りして解消——という定石に沿って全文を圧縮しました。
- 具体的な変更：ヒーロー見出しを課題起点（「保守が切れた基幹システムを、止めずに、まるごと引っ越し。」）に変更。リード・ステップ・FAQ・最終CTAの一文を短縮し、過剰な引用符・ダッシュ・三連の並列を削減。改訂した主要コピーは合計約1,541字→約1,248字（約19%短縮）。
- 不変：敬体、引っ越しのメタファー、単一CTA、Googleフォームのsrc、価格非掲載、実績の非捏造、オフショア非言及、絵文字なしはすべて維持しています。

## 何を・なぜ変えたか

### 1. シグネチャ「移送票」の強化（大胆さを集中させた1点）
- 天面に「旧→搬送→新」を表す3色のバーを追加し、票としての方向性を明確化しました。
- ステータス表示（IN TRANSIT）に点滅インジケータ、「5 / 5 完了」カウンタと進捗バーを新設。「診断だけでなく、移し終えるまで一気通貫でやり切る」を一目で伝える狙いです。
- 旧（かすれた破線・グレー）と新（くっきりした青・チェック）の対比を強めました。
- `<figure>` / `<figcaption>` に変更し、aria-labelで「イメージ図」であることを明示。実績の捏造と誤認を避けています。

### 2. タイポ・余白・情報階層
- 見出しに `text-wrap:balance`、本文に `pretty` を適用し、改行と行のバランスを改善しました。
- ボタンに最小高さ（52px）を設定し、押しやすさと視認性を確保。
- カードのホバー、チェックボックスのアイコン地を琥珀系に統一し、搬送の信号色で一貫性を出しました。

### 3. 中盤CTAの新設（FV直下／中盤／末尾の反復）
- 「選ぶ理由」の直後に無料診断バンドを追加し、長い本文の途中でも申し込みに移れるようにしました。
- すべてのCTAに `id` と `data-cta` / `data-cta-location`（header / hero / mid / footer）を付与。将来の計測に流用できます（計測タグ・SaaSは未追加）。

### 4. モバイル / 導線
- ヒーロー・中盤・フォームのボタンをモバイルで全幅化し、タップ領域を確保しました。
- `scroll-padding-top` を設定し、アンカー着地時に固定ヘッダーへ見出しが隠れないようにしました。
- フォーム直下に「表示されない場合は直接ご記入いただけます」リンクを追加（同一Googleフォームへの直リンク）。到達率の保険です。

### 5. アクセシビリティ / 構造
- `<main>` を導入し、各セクションに `aria-labelledby` を付与。装飾用SVG・図版要素は `aria-hidden` に。
- 見出し階層の飛びを解消（流れの項目を `<h4>` → `<h3>`、`<ol>/<li>` 化）。
- キーボードフォーカスの可視化と `prefers-reduced-motion` を維持・強化（進捗バーも低モーション時は静止）。

## 制約の順守（確認済み）
- CTAは「無料診断」の1つのみ。GoogleフォームのiframeはURLを含めて完全維持しています。
- 価格・料金表は非掲載（「診断は無料」の明示のみ）。グローバルナビなし（会社概要・フッターの最小リンクのみ）。
- 実績・お客様の声・導入社数・受賞歴などの捏造はなし。海外拠点・オフショアへの言及なし。ブランドは大和合同会社のみ。絵文字は不使用。

## オーナー側でのご対応をおすすめする事項（HTMLでは変更していません）

1. Googleフォームの必須項目の見直し（フォーム側の設定）
   - 現状は会社名／担当者名／電話番号／メール／お困りごとが必須で、入力ハードルはやや高めです。
   - 摩擦低減のご提案：まず必須を「会社名（またはお名前）」「メール」「お困りごと」程度に絞り、電話番号は任意に切り替えるとPoC段階の獲得数を伸ばしやすくなると考えられます（効果はテストで確認が必要です）。

2. iframe固定高の微調整（一部は実測で補正済み）
   - 実機のGoogleフォームを開いて測定した結果、フォーム本体は幅640px固定・全高1342pxでした。これに合わせPC側の高さを1280px→**1360px**へ補正しました（従来は約62px不足で、PC表示でフォーム内に縦スクロールが出る状態でした）。
   - スマホ（幅560px以下）は作業環境でウィンドウを縮小できず実測できなかったため、**1620px**（内側スクロールを避けるためやや高めの推定値）に設定しています。お手元のスマホで開き、フォーム下の余白が大きい／逆に内側スクロールが出る場合は、`@media(max-width:560px)` 内の `.form-embed iframe` の `height` を10〜40px単位で微調整してください。
   - フォームの設問を増減した際は、PC・スマホとも再調整が必要です（`.form-embed iframe` の `height` と、上記メディアクエリ内の値）。

3. 将来のアナリティクス導入
   - 各CTAに `data-cta`（header / hero / mid / footer）を付与済みです。GA4等を導入する際、この属性でクリック計測ができます（本ファイルには計測タグを入れていません）。

4. （任意）フォーム送信後の確認メッセージ
   - Googleフォーム側の完了メッセージに「診断は無料」「無理な営業はしない」旨を再掲すると、送信後の安心感が高まります。

5. 公開URL確定後のSEO仕上げ
   - 構造化データ（JSON-LD）の会社URLは `https://www.yamato.llc/` を記載しています。公開ドメインに合わせて調整してください。
   - GitHub Pages等で公開URLが決まったら、`<link rel="canonical">` と `og:url`、共有画像 `og:image`（1200×630px推奨）の追加をおすすめします（本ファイルにはURL未確定のため未設定）。
